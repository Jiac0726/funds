# 基基棒小程序端

这是为当前仓库补充的原生微信小程序端，入口目录是 `miniprogram/`。

## 已实现

- 自选基金列表
- 基金搜索添加
- 持有份额、成本净值编辑
- 实时估值、日收益、持有收益汇总
- 在“我的”中切换东方财富、天天基金估值或已配置的 Tushare 净值源
- 指数行情展示
- 基金详情、阶段表现、基金资料和最近净值列表
- 基于真实净值逐日回算最近三个月持仓金额、持有收益和收益率
- 本地记录历史建仓/加仓日期、份额、投入金额和成交净值
- 首页自选列表排序
- 支付宝基金持仓截图 AI 识别并生成可编辑导入表单
- OCR 文本兜底解析导入
- 本地存储自选配置
- 配置导出、导入和恢复默认

## 本地运行

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择本目录：`miniprogram/`。
4. 没有 AppID 时可先使用测试号/游客模式；正式发布前把 `project.config.json` 里的 `appid` 改成你的小程序 AppID。
5. 开发调试阶段可在微信开发者工具里勾选“不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书”。

## 正式发布前需要配置的 request 合法域名

正式上线建议部署 `server/` 后端代理，并在 `utils/config.js` 中把 `API_BASE_URL` 改成你的 HTTPS API 域名。

在微信公众平台的小程序后台配置：

- request 合法域名：`https://你的-api-域名`
- uploadFile 合法域名：`https://你的-api-域名`

开发阶段如果 `API_BASE_URL` 为空，会继续直连东方财富公开接口；支付宝截图导入需要配置后端域名和后端视觉/OCR 引擎后才能使用。

## 支付宝截图导入

入口在“我的 -> 支付宝截图导入”。

- 可一次选择多张支付宝“全部持有”或“资产详情”截图，小程序会逐张上传到你的后端，由后端调用视觉模型识别基金名称、代码、持有份额、成本净值和持有金额。
- 如果暂时没有视觉模型，可以把系统 OCR 或手动复制的文本粘贴到页面里解析；文本中包含 6 位基金代码时识别效果最好。
- 导入前会生成可编辑表单，用户可修改基金代码、名称、持有金额、份额和成本净值；确认后才合并到本机小程序缓存，已有基金会更新份额和成本，新基金会追加到自选。

## 上传发布

在微信开发者工具中打开 `miniprogram/` 后，点击“上传”，再到小程序后台提交审核。审核通过后即可发布。

## 说明

这个版本使用小程序本地存储保存自选基金，不依赖扩展端的 Chrome 同步数据。正式上线可使用仓库里的 `server/` 后端代理统一请求域名；后续如果要账号同步，可以在该后端上继续新增登录和用户配置接口。
